# Reference Open Model

This entry shows how to document open models with the same structure as commercial ones.

## What to include

- License and usage restrictions
- Weight availability
- Recommended inference stack
- Hardware requirements
- Fine-tuning notes

## Operational notes

Open models often change through:

- new checkpoints
- quantized variants
- third-party serving integrations
- community benchmark updates

Document those changes in the body and keep the frontmatter stable for filtering and listing.
