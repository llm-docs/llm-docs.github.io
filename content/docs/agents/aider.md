# Aider Guide

Aider is best treated as a practical terminal pair programmer for iterative edits inside a git repository rather than a full orchestration platform.

## Best Fit

- Small and medium implementation loops
- Users who want tight git feedback
- Teams that prefer lightweight coding assistance over heavy autonomy

## Rollout Advice

- Keep tasks scoped and reviewable
- Preserve clean git discipline around each session
- Use it as a pair programming layer, not a substitute for code ownership
- Benchmark it on repetitive engineering tasks first
