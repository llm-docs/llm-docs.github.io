# OpenCode Guide

OpenCode is a good option when terminal ergonomics, open-source control, and model-provider flexibility are first-order requirements.

## Best Fit

- Teams that want to avoid provider lock-in
- Developers who run several agent sessions in parallel
- Terminal-heavy workflows that benefit from LSP-aware context

## Rollout Advice

- Choose a standard provider setup before scaling usage
- Define when multi-session workflows are appropriate
- Keep prompt and tool policies consistent across sessions
- Compare performance against a single-vendor baseline
